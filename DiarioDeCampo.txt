Hoy estudiando por mi cuenta aprendi teoría sobre HTML y CSS, ademas de estudiar un poco de ingles.
Estudiando HTML aprendí a realizar una plantilla de una pagina, en la cual tienes que ingresar una etiqueta !DOCTYPE, luego una etiqueta html con un
atributo el cual sera el idioma de la pagina, luego, adentro de esa etiqueta html, debemos ingresar el head (donde tendremos metadatos, el titulo y el charset uft-8)
y el body (Donde estara los headers y los parrafos).

En HTML aprendí a realizar un boton, un formulario, los diferentes tipos de formularios (text, password, range, date, color, mail, submit, etc), como añadir multimedia
tanto como imagenes, videos o audios. Ademas de esto, aprendí a introducir footer's y la forma correcta de escribir codigo HTML.
Uno debe escribir bien tabulado, cuidar la ortografía, poner bien las etiquetas, no escribir codigo espaguetti, ser coherente con lo que se escribe y no agrupar headers.

En CSS aprendí a poner un poco de estilo a el texto, como seleccionar cada caja o cada elemento, que se puede hacer tanto, seleccionando de forma universal con un *
Tambien esta la forma de seleccionar con clases, que es poniendo un punto y luego el nombre de la clase , tambien existe la forma de seleccionar por ID (LA ID DEBE 
SER UNICA PARA CADA ELEMENTO, PARA LUEGO TRABAJAR DE FORMA CORRECTA CON CADA ELEMENTO CON JAVASCRIPT), que se realiza primero introduciendo un numeral # y luego el ID
del elemento a estilizar. Tambien se puede seleccionar elementos especificos que estan adentro de otros, por ejemplo, si yo quiero estilizar solo h3 que esten adentro 
de la clase achivements deberia escribirlo de la siguiente manera: .achivements h3{ style:atribute; } y así estilizaria solo los elementos h3 que esten dentro de la 
clase achivements.
La siguiente forma de seleccion es con pseudo-clases, que son clases ya hechas por CSS que describen un evento o accion, tal como la pseudo-clase hover, si introduces
el siguiente codigo h1:hover {color:red;} estarías estilizando todos los h1 tal que cuando les pases el cursos por encima se volveran de color rojo.

Estos dias he estado estudiando HTML y CSS, he aprendido la semantantica de la escritura de HTML, como seria el formato estandar de una pagina web, estaria dividida
en head, donde va el titulo y los metadatos, luego iria el body, donde estaria todo lo que el usuario vería, allí, primero va un header y/o una barra de navegacion
la cual va a ser comun a todas las paginas relacionadas. Luego del header va el section, que es donde estara ubicado, todo el texto o contenido de la pagina, alli 
se deben dividir las secciones en articulos, y a su vez, en caso de que sea necesario, deberan estar dentro de una etiqueta div para poder organizar en caja mejor el contenido.
Luego de el section se puede insertar una etiqueta aside, que sería una barra lateral para mostrar informacion secundaria o una publicidad, aunque en HTML esto escrito queda
sin buena estetica y queda uno abajo del otro, se puede arreglar con CSS, con CSS podemos mover las cajas (cada etiqueta es una caja, y hay cajas dentro de cajas) y
ponerlo en el lugar indicado, como indica aside, debería ir a un lado, y con CSS lo ponemos a un lado.

Aprendí tambien a usar de mejor manera las listas, tanto las ordenadas como las desordenadas, aprendí a la etiqueta input, la cual tiene atributo class que podemos indicar
diferentes tipos de inputs, como text, password, range, date, range, color y submitt esto tambien lo podemos editar con CSS. tambien esta la etiqueta form, que allí podemos hacer un
formulario en el cual poner distintos inputs, y, enviarlos al backend/base de datos para el procesamiento, osea que, iria una etiqueta form y adentro de ella los distintos inputs y,
para estilizar mejor, seria una buena practica insertar la etiqueta form adentro de un div.

Algo muy importante para la programacion web, es el SEO, esto ese es el posicionamiento que le da los navegadores a nuestra pagina web, esto se mejora y aumenta escribiendo
HTML semanticamente correcto, usar el atributo alt en las imagenes (que esto es ponerle un texto ) y tambien afectan bastante los metadatos,siendo la descripcion, el titulo, y si
es apto para movil, si esta traducido, si va a ser indexado (Ser indexado significa aparecer en la busqueda de google, si no es indexada la pagina no aparecera en 
el indice de los navegadores)

Luego del Aside, <em>que es un gran elemento en la pagina, ya que cada uno puede darle el aspecto e importancia que quiere</em> va el footer, que es el pie de pagina, no debe
ser necesariamente igual en todas las paginas, pero es comun que en la pagina principal tenga datos bastante importantes, tanto como los derechos de autor.SS
